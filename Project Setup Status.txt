POST /api/users/login — returns JWT token

GET /api/users/profile — returns user info for logged-in users

🔹 Features
Passwords hashed with bcryptjs

JWT-based login system

Middleware protect checks for valid token

✅ Role-Based Design
Allowed user roles: admin, tutor, student

JWT token includes both id and role

You (as admin) used token to create classes

✅ Class Management (for Admin)
🔹 Routes
POST /api/classes/create — admin-only route to create a class

GET /api/classes — public route to view all classes

🔹 Features
Class model with fields: name, subject, schedule, tutor

Classes are linked to tutors using their MongoDB _id

Implemented .populate("tutor", "name email") (you can confirm this later)
